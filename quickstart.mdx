Get your Astro Services application running quickly with this streamlined guide. You'll go from zero to a fully functional streaming services platform in just a few minutes.

## Prerequisites

Before you begin, ensure you have the following installed:

<CardGroup cols={2}>
  <Card title="Node.js" icon="node-js" href="https://nodejs.org/">
    Version 18 or higher required
  </Card>
  <Card title="MongoDB Atlas" icon="database" href="https://www.mongodb.com/atlas">
    Free tier account is sufficient
  </Card>
  <Card title="Git" icon="git" href="https://git-scm.com/">
    For cloning the repository
  </Card>
  <Card title="Package Manager" icon="box">
    npm, yarn, or pnpm
  </Card>
</CardGroup>

<Note>
  **Quick Check**: Verify Node.js is installed by running `node --version` in your terminal. You should see v18.0.0 or higher.
</Note>

## Quick Setup

<Steps>
  <Step title="Clone the Repository">
    Clone the Astro Services repository to your local machine:

    ```bash
    git clone https://github.com/luisllatas-dev/proyecto_streaming.git
    cd Proyecto_Astro
    ```
  </Step>

  <Step title="Install Backend Dependencies">
    Navigate to the backend directory and install dependencies:

    <CodeGroup>
      ```bash npm
      cd astro_backend
      npm install
      ```

      ```bash yarn
      cd astro_backend
      yarn install
      ```

      ```bash pnpm
      cd astro_backend
      pnpm install
      ```
    </CodeGroup>
  </Step>

  <Step title="Configure Environment Variables">
    Create a `.env` file in the `astro_backend` directory with your MongoDB connection:

    ```bash
    touch .env
    ```

    Add your MongoDB Atlas URI to the `.env` file:

    ```env
    PORT=5000
    MONGODB_URI=mongodb+srv://<usuario>:<password>@<cluster>.mongodb.net/astroservicesdb
    ```

    <Warning>
      Replace `<usuario>`, `<password>`, and `<cluster>` with your actual MongoDB Atlas credentials.
    </Warning>
  </Step>

  <Step title="Start the Backend Server">
    Launch the backend API server:

    <CodeGroup>
      ```bash npm
      npm run dev
      ```

      ```bash yarn
      yarn dev
      ```

      ```bash pnpm
      pnpm dev
      ```
    </CodeGroup>

    You should see:
    ```
    MongoDB conectado: <your-cluster>.mongodb.net
    Servidor corriendo en http://localhost:5000
    ```
  </Step>

  <Step title="Seed the Database">
    Open a new terminal window and populate the database with sample services:

    <CodeGroup>
      ```bash npm
      cd astro_backend
      npm run seed
      ```

      ```bash yarn
      cd astro_backend
      yarn seed
      ```

      ```bash pnpm
      cd astro_backend
      pnpm seed
      ```
    </CodeGroup>

    You'll see:
    ```
    MongoDB conectado para seeding...
    Colección de servicios limpiada.
    6 servicios insertados correctamente.
    Conexión cerrada. Seed completado.
    ```

    <Note>
      This creates 6 sample streaming services including Netflix Premium, Disney+ Premium, HBO Max, Spotify Premium, IPTV Full HD, and Amazon Prime Video.
    </Note>
  </Step>

  <Step title="Install Frontend Dependencies">
    Open another terminal, navigate to the frontend directory, and install dependencies:

    <CodeGroup>
      ```bash npm
      cd astro_services
      npm install
      ```

      ```bash yarn
      cd astro_services
      yarn install
      ```

      ```bash pnpm
      cd astro_services
      pnpm install
      ```
    </CodeGroup>
  </Step>

  <Step title="Start the Frontend Application">
    Launch the React application:

    <CodeGroup>
      ```bash npm
      npm run dev
      ```

      ```bash yarn
      yarn dev
      ```

      ```bash pnpm
      pnpm dev
      ```
    </CodeGroup>

    The application will be available at `http://localhost:5173`
  </Step>

  <Step title="Access the Application">
    Open your browser and navigate to:

    ```
    http://localhost:5173
    ```

    You should see a grid of streaming service cards displaying the 6 sample services you seeded!
  </Step>
</Steps>

## Verify Your Setup

Test that everything is working correctly:

<Tabs>
  <Tab title="Frontend">
    Visit `http://localhost:5173` and verify you see:
    - A header with "Astro Services"
    - A grid of 6 service cards
    - Each card showing an image, title, price, discount, and description
  </Tab>

  <Tab title="Backend">
    Test the API health endpoint:

    ```bash
    curl http://localhost:5000/api/health
    ```

    Test fetching all services:

    ```bash
    curl http://localhost:5000/api/services
    ```

    You should receive a JSON response with all 6 services.
  </Tab>
</Tabs>

## Quick Troubleshooting

<AccordionGroup>
  <Accordion title="Backend won't start - MongoDB connection error">
    - Verify your MongoDB Atlas URI is correct in the `.env` file
    - Ensure your IP address is whitelisted in MongoDB Atlas (Network Access)
    - Check that your database user credentials are correct
    - Confirm your internet connection is active
  </Accordion>

  <Accordion title="Frontend can't connect to backend">
    - Ensure the backend is running on port 5000
    - Check that no firewall is blocking port 5000
    - Verify CORS is properly configured (it's enabled by default in the backend)
  </Accordion>

  <Accordion title="Seed command fails">
    - Make sure the backend `.env` file has a valid MONGODB_URI
    - Ensure MongoDB Atlas cluster is running and accessible
    - Try running the backend first with `npm run dev` to test the connection
  </Accordion>
</AccordionGroup>

  For more comprehensive troubleshooting, see the [Installation Guide](/installation#troubleshooting).
</Note>
