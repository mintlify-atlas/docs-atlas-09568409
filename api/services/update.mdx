---
title: Update Service
description: Update an existing service
---

Updates an existing service with the provided information. Only the fields included in the request body will be updated.

## Endpoint

```
PUT /api/services/:id
```

## Request

### Path Parameters

<ParamField path="id" type="string" required>
  The MongoDB ObjectId of the service to update. Must be a valid 24-character hexadecimal string.
</ParamField>

### Headers

```
Content-Type: application/json
```

### Body Parameters

All fields are optional. Only include the fields you want to update.

<ParamField body="imagen" type="string">
  URL of the service image. Will be trimmed of whitespace if provided.
</ParamField>

<ParamField body="titulo" type="string">
  Service title or plan name. Maximum length of 120 characters. Whitespace is automatically trimmed.
</ParamField>

<ParamField body="precio" type="number">
  Service price. Must be greater than or equal to 0 if provided.
</ParamField>

<ParamField body="descuento" type="number">
  Discount percentage. Must be between 0 and 100 if provided.
</ParamField>

<ParamField body="descripcion" type="string">
  Service description. Maximum length of 500 characters. Whitespace is automatically trimmed.
</ParamField>

### cURL Example

```bash
curl -X PUT http://localhost:5000/api/services/507f1f77bcf86cd799439011 \
  -H "Content-Type: application/json" \
  -d '{
    "precio": 12.99,
    "descuento": 20
  }'
```

### JavaScript Example

```javascript
const serviceId = '507f1f77bcf86cd799439011';
const updates = {
  precio: 12.99,
  descuento: 20
};

const response = await fetch(`http://localhost:5000/api/services/${serviceId}`, {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(updates)
});

const updatedService = await response.json();
```

## Implementation

Controller function at `/home/daytona/workspace/source/astro_backend/src/controllers/serviceController.js:41-54`:

```javascript
export const updateService = async (req, res) => {
  try {
    const service = await Service.findByIdAndUpdate(req.params.id, req.body, {
      new: true,
      runValidators: true,
    });
    if (!service) {
      return res.status(404).json({ message: 'Servicio no encontrado' });
    }
    res.json(service);
  } catch (error) {
    res.status(400).json({ message: 'Error al actualizar el servicio', error: error.message });
  }
};
```

## Update Options

The update operation uses the following Mongoose options:

- **new: true** - Returns the updated document rather than the original
- **runValidators: true** - Runs model validation on the update operation, ensuring all schema constraints are enforced

## Validation Rules

The same validation rules apply as when creating a service:

| Field | Validation Rules |
|-------|------------------|
| `imagen` | Trimmed, must not be empty if provided |
| `titulo` | Trimmed, max 120 characters if provided |
| `precio` | Minimum value 0 if provided |
| `descuento` | Min 0, max 100 if provided |
| `descripcion` | Trimmed, max 500 characters if provided |

## Partial Updates

You can update only specific fields by including only those fields in the request body. Unspecified fields will remain unchanged.

### Example: Update Only Price

```bash
curl -X PUT http://localhost:5000/api/services/507f1f77bcf86cd799439011 \
  -H "Content-Type: application/json" \
  -d '{"precio": 14.99}'
```

## Notes

- The `updatedAt` timestamp is automatically updated by MongoDB
- You cannot update the `_id`, `createdAt`, or `__v` fields
- All validation rules are enforced even for partial updates
- Invalid ObjectId formats will result in a 400 error
- Returns the complete updated document, not just the changed fields
