Database seeding allows you to quickly populate your MongoDB database with sample services for development and testing. The seed script clears existing data and inserts predefined service entries.

## Quick Start

Run the seed script from the backend directory:

```bash
cd astro_backend
npm run seed
```

This command will:
1. Connect to your MongoDB database
2. Clear all existing services
3. Insert 6 sample services
4. Close the connection

## Seeding Process

<Steps>
  <Step title="Configure DNS">
    Sets Google DNS servers to resolve MongoDB Atlas SRV records:
    
    ```javascript
    dns.setServers(['8.8.8.8', '8.8.4.4']);
    ```
    
    This ensures reliable DNS resolution for MongoDB Atlas connections.
  </Step>

  <Step title="Connect to MongoDB">
    Establishes connection using the URI from environment variables:
    
    ```javascript
    await mongoose.connect(process.env.MONGODB_URI);
    console.log('MongoDB conectado para seeding...');
    ```
    
    Make sure your `.env` file has the correct `MONGODB_URI`.
  </Step>

  <Step title="Clear Existing Data">
    Removes all existing services to start fresh:
    
    ```javascript
    await Service.deleteMany({});
    console.log('Colección de servicios limpiada.');
    ```
    
    **Warning**: This deletes all services in the database.
  </Step>

  <Step title="Insert Sample Services">
    Inserts the 6 predefined services:
    
    ```javascript
    const created = await Service.insertMany(sampleServices);
    console.log(`${created.length} servicios insertados correctamente.`);
    ```
  </Step>

  <Step title="Close Connection">
    Properly closes the MongoDB connection and exits:
    
    ```javascript
    await mongoose.connection.close();
    console.log('Conexión cerrada. Seed completado.');
    process.exit(0);
    ```
  </Step>
</Steps>

## Service Schema Structure

Each service follows the Mongoose schema defined in `Service.js`:

| Field | Type | Required | Validation |
|-------|------|----------|------------|
| `imagen` | String | Yes | Valid URL, trimmed |
| `titulo` | String | Yes | Max 120 characters, trimmed |
| `precio` | Number | Yes | Minimum 0 |
| `descuento` | Number | No | 0-100, defaults to 0 |
| `descripcion` | String | Yes | Max 500 characters, trimmed |

Timestamps (`createdAt`, `updatedAt`) are automatically added by Mongoose.

## Running the Seed Script

### Prerequisites

Ensure you have:

1. **MongoDB connection configured** in `.env`:
   ```env
   MONGODB_URI=mongodb+srv://<usuario>:<password>@<cluster>.mongodb.net/astroservicesdb
   ```

2. **Dependencies installed**:
   ```bash
   cd astro_backend
   npm install
   ```

### Execute Seeding

```bash
npm run seed
```

### Expected Output

```
MongoDB conectado para seeding...
Colección de servicios limpiada.
6 servicios insertados correctamente.
Conexión cerrada. Seed completado.
```

## Resetting the Database

To completely reset and reseed the database:

<Steps>
  <Step title="Stop Running Servers">
    Make sure no backend server is running to avoid connection conflicts.
  </Step>

  <Step title="Run Seed Script">
    ```bash
    npm run seed
    ```
    
    This clears all existing services and inserts fresh sample data.
  </Step>

  <Step title="Restart Backend">

## Production Considerations

<Warning>
  The seed script **deletes all existing services** before inserting new ones. Never run this in production unless you intend to reset all data.
</Warning>

For production environments:

1. **Create a separate seed script** that doesn't delete existing data
2. **Use migrations** for data changes instead of seeding
3. **Backup data** before running any seed operations
4. **Use different sample data** to avoid copyright issues with service names

## Advanced Seeding

### Conditional Seeding

Only seed if the database is empty:

```javascript
const seedDB = async () => {
  try {
    await mongoose.connect(process.env.MONGODB_URI);
    
    const count = await Service.countDocuments();
    if (count > 0) {
      console.log('Database already has data. Skipping seed.');
      process.exit(0);
    }
    
    const created = await Service.insertMany(sampleServices);
    console.log(`${created.length} servicios insertados correctamente.`);
    
    await mongoose.connection.close();
  } catch (error) {
    console.error('Error durante el seed:', error.message);
    process.exit(1);
  }
};
```

### Incremental Seeding

Add new services without deleting existing ones:

```javascript
// Remove the deleteMany() call
// await Service.deleteMany({});

// Insert new services
const created = await Service.insertMany(sampleServices);
```

### Seeding with Relationships

If you add related data models (users, categories, etc.), seed them in order:

```javascript
// Seed categories first
const categories = await Category.insertMany(sampleCategories);

// Then seed services with category references
const services = await Service.insertMany(
  sampleServices.map(s => ({
    ...s,
    category: categories[0]._id
  }))
);
```
